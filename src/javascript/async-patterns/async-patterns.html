Async Patterns in JS

Asynchronous operations allow Javascript to perform non-blocking tasks. These include: fetching data, reading file and background timers - without blocking code execution. These operations are essential for making the applications more responsive and smooth.

Three primary patterns include:

  • Callback funcitons
  • Promises
  • async and await

Callback function

Any function passed into another function as an argument, which gets executed once an operation finishes, is a callback function.

There is no specific keyword like callback that designate a funtion as callback function. Functions in JS are first-class citizens and are treated as any other variables like string or numbers. Typically the name 'callback' is used for such functions.

While callback is functional, it is discouraged for complex async operations because nested calls can become difficult to manage and commonly called "callback hell".

function greetings(user, callbackOrAnyName) {
    console.log("Hello "+user);
    greet();
}

// Any function
function greet() {
    console.log("Nice to meet you!!");
}

// Passing callback function
greetings("Mohit", greet);


// ---- Using anonymous function ----
function wait(callback) {
    setTimeout(() => {
        callback("function executed!");
    }, 1000);
}

wait((message) => {
    console.log(message);
});

Promises

  • A promise is an object representing a value that may or may not be available of an asyncronous operation.
  • It act as a placeholder for a value that may no be available immediately but will be either successfully returned(resolved) or result in an error(rejected).
  • A promise represents eventual completion or failure of an async operation.
  • It has three states - Pending, Fulfilled and Rejected.
  • It allow chaning of operations using .then() and .catch(), helping flattening the code.
  • Promises are built-in part of JS, while frameworks like Angular primarily uses RxJS Observable for async tasks.

While promise chaining using then() is great, It can still get "wordy". That is where async / await comes in.

// Using Promise
// fetch immediately sends an http request and returns a promise without waiting for the request to complete.
const myPromise = fetch("https://jsonplaceholder.typicode.com/posts/1");

// the immediate status of promise will be pending
console.log("Current status: ", myPromise);

// Promise chaining to decide on data transformation.
myPromise.then((response) => {
    return response.json(); // wait for data to parse and returns a new promise for the next then() to process.
})
.then((data) => {
    console.log("Data received: ", data.title);
}).catch((error) => {
    console.error("Connection faile: ", error);
}).finally(() => {
    console.log("Operation finished!");
});

Async / await

Async / await are keywords allow you to write asynchronous code in a clean and synchronous-like style.

  • Easy to read and maintain compared to promise chaining methods: .then() and .catch().
  • Built on top of JS Promise.
  • Placing 'async' before a function makes it asynchronous and it automatically wraps and return a Promise.
  • await pauses the execution of async until the promise is resolved or rejected.
  • await can only be used inside async function.
  • await doesn't block the main thread, only yields control to rest of the code and events.
// Async and Await
async function getPost() {
    try {
        const response = await fetch("https://jsonplaceholder.typicode.com/posts/1");
        const responsePromise = await response.json();
        console.log("Using async / await: ",responsePromise.title);
    }catch(error) {
        console.log(error);
    }
}
getPost();
