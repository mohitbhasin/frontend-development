Dynamically managing DOM elements

Append vs Append Child

append() can be used to add an element in Node object form. Or add a DOMString (text).

.append() accepts multiple argument.

// append(Node_object) for parent <ul>.append(<li>) converts to:
// <ul><li>...</li></ul>

// append(String) for parent <ul>.append("Banana") converts to:
// <ul>Banana></ul>
    Items:

appendChild() accepts single argument of type Node object.

Similarly, there are remove() and removeChild() methods to remove objects and DOMString.

remove(node) is precise and doesn't care about the parent.

removeChild(child node) requires the parent of the node to be removed.

Cloning and Replacing Nodes

cloneNode(true/false) method allows to create a copy of an element. True returns deep copy and False returns shallow copy.

replaceChild() method replaces existing node with a new one.

Using querySelector() within div object (not on document.querySelector()) to find the cloned para and repalcing it with the new one.

insertBefore()

Syntax: parentElement.insertBefore(newNode, childNode)

To add the new node as the first item, use parent.firstChild. To add after a specific child, use child.nextSibling.
