Modules in javascript

Module organizes code in reusable files, preventing naming conflicts and easier maintenance. It uses ES(ECMAScript) Modules, which uses import and expoert statement.

To use modules in html, you must specify script tag with type module and src to module's js file.

(Modules must be served over http(s). Running locally using file:// protocol will throw CORS blocked error.)

Module Scope

Variables and functions inside a module are inaccessable outside, unless explicitly exported.

Import/Export

Use export to expose variables, functions and classes to other files. Use import to use exposed functionalities of modules.

export const PI = 3.141;
export function add(a, b) {
  return a+b;
}
import {
  PI, add
} from './sample-module.js';

//import as alias
import { add as sum } from './sample-module.js'

console.log(PI);
console.log(add(4,9));

Types of exports and imports

(A module can have both named and default exports)

    Named exports:
  • Allows a module to export multiple variables, functions or classes using export before their decleration.
  • No limit on the number of exports that can be made.
  • Must use the exact name or 'as' to import named exports
  • Use curly braces - import { PI } from './module.js'
  • Purpose is to use with utility functions, constants and helper tools
    Default exports:
  • Allows only one default export per module.
  • Can be imported with any name.
  • No curly braces required - import PI_VAL from './module.js'
  • Purpose is to export main thing that represents class or component.
const subject = "Mathematics";
export default subject;
import course from './sample-module.js';
console.log(course);
